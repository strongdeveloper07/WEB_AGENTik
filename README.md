
# Кроссплатформенный Web-Агент для работы с кардиографом

## Общее описание проекта

Проект представляет собой распределённую систему для взаимодействия с медицинским устройством (кардиографом), подключённым к рабочему ПК медсестры.

Система позволяет:

- Авторизоваться в веб-приложении через браузер
- Привязать учётную запись к конкретному устройству (ID кардиографа)
- Загружать данные с устройства на сервер
- Выполнять минимальное управление устройством через веб-интерфейс

---

## Архитектура системы

Система состоит из трёх компонентов:

1. Локальный агент (C++)
2. Сервер (Backend)
3. Веб-приложение (Frontend)

### Общая схема (примерно на данном этапе)

```
[Кардиограф]
↓
[ПК медсестры]
↓
[Локальный агент (C++)]
↓ HTTP/HTTPS
[Сервер]
↓
[Веб-приложение]
↓
[Браузер медсестры]
```

### Распределение ответственности

- Локальный агент взаимодействует с устройством и файловой системой.
- Сервер управляет задачами, хранит данные и обеспечивает API.
- Веб-приложение предоставляет пользовательский интерфейс.

---

## Функциональные требования

### Локальный агент

Агент должен:

1. Работать на Windows, Linux и macOS.
2. Работать в фоновом режиме.
3. Подключаться к серверу по HTTP/HTTPS.
4. Регистрироваться на сервере с уникальным идентификатором (UID).
5. Периодически запрашивать задачи (polling).
6. Выполнять полученные задачи.
7. Отправлять результаты выполнения на сервер.
8. Загружать файлы на сервер.
9. Вести журнал действий и ошибок.

### Сервер

Сервер должен:

1. Регистрировать агентов.
2. Управлять сессиями.
3. Создавать и выдавать задачи.
4. Принимать результаты выполнения.
5. Принимать и сохранять загруженные файлы.
6. Предоставлять API для веб-приложения.

### Веб-приложение

Веб-приложение должно позволять:

1. Выполнять авторизацию пользователя.
2. Привязывать пользователя к ID устройства.
3. Запускать процесс загрузки данных.
4. Отображать статус выполнения задач.

---

## Нефункциональные требования

- Кроссплатформенность.
- Низкое потребление ресурсов.
- Поддержка одновременной работы нескольких агентов.
- Устойчивость к сетевым сбоям.
- Модульная и расширяемая архитектура.
- Читаемый и аккуратно оформленный исходный код.

---

## Конфигурация агента

Агент должен использовать конфигурационный файл, содержащий:

- `uid` — уникальный идентификатор агента
- `server_url` — адрес сервера
- `poll_interval` — интервал опроса сервера
- `tasks_directory` — папка для хранения задач
- `results_directory` — папка для результатов
- `log_file` — путь к журналу
- параметры окружения (при необходимости)

---

## API взаимодействия (Агент ↔ Сервер)

Все запросы выполняются по HTTP/HTTPS с использованием JSON.

Базовый путь: 
``` /api/v1/ ```

---

### 1. Регистрация агента

**POST** `/api/v1/agents/register`

Запрос:

```json
{
  "uid": "agent-uuid",
  "platform": "linux",
  "hostname": "pc-01",
  "version": "1.0.0"
}
```
Ответ:
```json
{
  "status": "ok",
  "session_token": "string",
  "poll_interval": 10
}
```


## Состав системы

1. Server: Центральный узел на базе Node.js, отвечающий за хранение данных в базе SQLite и обработку запросов.
2. Agent: Высокопроизводительное приложение на C++, предназначенное для отслеживания изменений в папке врача и передачи информации на сервер.
3. Web UI: Клиентская часть (HTML/JS) для доступа к системе через браузер.

## Инструкция по запуску и эксплуатации

### 1. Подготовка сервера

1. Откройте терминал или командную строку.
2. Перейдите в директорию сервера:
   cd server
3. Установите программные зависимости:
   npm install
4. Запустите сервер:
   node index.js

После этого сервер будет ожидать подключений по адресу http://localhost:3000.

### 2. Сборка и запуск Агента
Агент компилируется из исходного кода непосредственно на устройстве пользователя.

1. Откройте новое окно терминала.
2. Перейдите в директорию агента:
   cd agent
3. Создайте рабочую директорию для сборки:
   mkdir build
   cd build
4. Выполните конфигурацию и компиляцию:
   cmake ..
   cmake --build .
5. Запустите скомпилированный файл:
   ./cardio_agent

Примечание: Агент автоматически подключается к локальному серверу. Убедитесь, что сервер был запущен ранее.

### 3. Использование веб-интерфейса
Для просмотра данных и управления пользователями:

1. Перейдите по адресу http://localhost:3000
2. Пройдите процедуру авторизации или регистрации.
3. В главном меню вы сможете просматривать список подключенных устройств и карточки мониторинга.

## Техническая структура проекта

* /server — исходный код серверной части и файл базы данных database.sqlite.
* /agent — исходный код мониторинга на языке C++.
* /web — статические файлы интерфейса (HTML, CSS, JavaScript).

---

## Важные примечания по эксплуатации

* Сетевые настройки: По умолчанию все компоненты настроены на работу через локальную сеть (localhost). Для работы через интернет потребуется настройка IP-адресов в конфигурационных файлах.
* База данных: Система использует SQLite, которая не требует отдельной установки сервера баз данных. Файл создается автоматически при первом запуске сервера.
* Режим работы: Агент должен оставаться запущенным в терминале для непрерывного сбора данных.
